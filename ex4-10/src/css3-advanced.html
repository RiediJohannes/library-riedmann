Container Queries

Container Queries is one of the latest and highly anticipated additions to the CSS language. Unlike traditional media queries which respond to the dimensions of the viewport, Container Queries allow styles to adapt based on the dimensions of an element's container (parent). This means that you can apply styles to an element depending on how much space it has available, enabling more precise and context-aware layouts. For example, you could dynamically adjust the font size inside a card based solely on the width of its parent element, not based on the overall screen width. Therefore, the layout of the webpage can be changed more freely as the content of the cards will adapt automatically to the space given.

CSS Nesting

CSS nesting allows you to nest CSS selectors within one another, similar to how they are structured in the HTML document. This makes it easier to write modular and organized stylesheets, as selectors can be grouped together based on their relationships in the HTML markup. For instance, you could define styles for your <a> tag within an existing selector for the <nav> tag, thereby scoping those styles only to links inside a navbar. Previously, you had to write a separate CSS block with the selector nav a to achieve the same behaviour. The addition of this new feature thus massively enhances code readability and maintainability. In fact, it has been one of the main "selling points" of the popular CSS preprocessor SASS/SCSS.
