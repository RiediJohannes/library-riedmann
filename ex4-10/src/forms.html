Verdict

When it comes to using plain CSS or a CSS library like Bulma, both solutions enable the creation of fairly modern and responsive HTML forms. However, there are some notable differences in the developer experience I want to highlight.

Bulma is much more beginner friendly, as it does not require an extensive knowledge of CSS properties and their behaviour. The predetermined Bulma classes substantially reduce the amount of code that has to be written to achieve a decent styling. Thanks to the awesome documentation on Bulma's website, the implementation is very straight-forward with minimal roadblocks.

Although, when you do run into a roadblock, it is often a very severe one. Using classes instead of custom CSS makes the styling process much more intrasparent which sometimes leads to a lot of frustration on my side to reach the desired look. Also, the use of Bulma classes often requires a staggering number of div containers which I am really not a fan of. In my opinion, all the extra divs substantially hamper the readability of the HTML code.

In conclusion, CSS libraries like Bulma are powerful tools to create modern and responsive websites in a short period of time. Especially after you made yourself familiar with a particular library, incredibly fast implementation times can be achieved. However, these libraries are not with their downsides. While it is easy to implement styles that the web has already seen thousands of times, creating elements with your personal vision and character can be exhausting at times.
In my opinion, CSS libraries are a wonderful tool to implement professional-looking webpages quickly, especially for CSS beginners. But people who are already familiar with CSS tricks might be better off writing their own CSS (preferrably SCSS) instead of wasting time "fighting against" the library.
